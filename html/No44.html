AugMix: A Simple Data Processing Method to Improve Robustness and Uncertainty

AugMix 是一種數據增強技術，旨在提升模型的穩健性 (Robustness) 和對不確定性 (Uncertainty) 的處理能力。

核心思想：將多種簡單的增強操作隨機鏈接，再與原始圖像混合。

通過 Jensen-Shannon Divergence Loss 保持原始圖像與增強圖像預測的一致性。

目標：使模型在面對各種數據擾動時，仍能做出一致且準確的預測。

AugMix 流程簡介

  1. 對原始圖像進行多次（例如3次）不同的基礎增強操作 (Augmentation Operations)。
  2. 將這些增強後的圖像通過隨機權重 (w1, w2, w3, 總和為1) 進行凸組合 (Convex Combination)。
  3. 將組合後的圖像再與原始圖像以特定權重 (w'2) 混合，產生一個 AugMix 圖像。
  4. 重複此過程 N 次，得到 N 個不同的 AugMix 圖像。
  5. 將原始圖像和這 N 個 AugMix 圖像輸入模型，計算其預測的平均值。
  6. Loss Function 包含 JS Divergence (確保預測一致性) 和標準分類 Loss。

提升穩健性

Improved Robustness

有效應對數據漂移和損壞 (Data Corruption)。

過度參數化 Overparameterization

現代深度學習模型通常具有遠超訓練數據量的參數。

  • 傳統觀點 (Classical Regime): 模型複雜度增加到一定程度後，測試風險上升 (Overfitting)。
  • 現代觀點 (Modern Interpolating Regime): 當模型極度過度參數化時，測試風險可能再次下降，出現 "Double Descent" 現象。
  • L1/L2 Regularization 在經典情境有效，但在過度參數化模型中可能不是最佳解。
  • 數據增強 (如 AugMix) 和使用未標註數據可視為一種對抗過度參數化的 Regularization 方法。

自監督學習概念 Self-supervised Concept

核心思想：利用數據自身的信息來創建監督信號，從而學習有用的表示 (Representation)。

Π Model (Pi Model)

對同一輸入應用兩種不同的隨機增強 (或 Dropout)，通過同一網絡得到兩個預測，並要求這兩個預測盡可能一致。

Temporal Ensembling

將過去多個 Epoch 的模型預測的指數移動平均 (Exponential Moving Average) 作為 "Teacher" 的輸出，當前 Epoch 的模型輸出作為 "Student"，要求兩者一致。目標是產生更穩定的監督信號。

關鍵：一致性正則化 (Consistency Regularization)

對比學習 Contrastive Learning

目標：學習一個表示空間，使得相似的樣本在該空間中距離相近，不相似的樣本距離遙遠。

核心原則

"We need to tell the neural network that 'these are the same thing'." 即，同一物體的不同視角 (Augmentations) 應該被映射到表示空間中的相近位置。

SimCLR

  • 對同一圖像產生兩個增強視圖 (Positive Pair)。
  • Batch內其他圖像的增強視圖視為 Negative Pairs。
  • 目標：拉近 Positive Pair 的表示，推遠 Negative Pairs 的表示。
  • 使用 Projection Head 和 Cosine Similarity。
  • Tips: 大 Batch Size (e.g., 4096, 8192) 效果好，常配合 LARS 優化器。

MOCO (Momentum Contrast)

  • 使用 Momentum Encoder 提供更穩定的 Negative Keys。
  • Momentum Encoder 的權重是 Query Encoder 的指數移動平均。
  • 避免了對極大 Batch Size 的依賴。
  • InfoNCE Loss。
  • Tips: BN 可能洩漏信息，需謹慎使用或配合 Shuffling。

對比學習提升 OOD 檢測 Contrastive Learning Improve OOD Detection

Out-of-Distribution (OOD) Detection 指的是識別出與訓練數據分佈不同的樣本。

通過 Contrastive Learning，模型學習到的 In-Distribution 數據表示會更緊湊。

這使得 OOD 樣本在表示空間中更容易被區分出來，因為它們的 Log Probability 通常較低。

AUROC (Area Under ROC Curve) 指標從 0.67 提升到 1.00，顯示了顯著的改進。

檢測方法:

  • Density Estimation: 估計樣本的概率密度。
  • Confusion Log Probability (CLP): 類似於 SimCLR 的思路，利用模型對不同類別的預測混淆程度。
